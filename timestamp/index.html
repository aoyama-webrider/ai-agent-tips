Claude Code Hook UserPromptSubmit Cursor

Cursor内のClaude Codeチャットに日時を表示する

Claude Codeをすでに使っている人向け

なぜ日時表示が必要なのか

Cursor内のClaude Codeチャットでは、各メッセージの送信時刻が画面に出ません。少し前のやり取りを遡るときに 「これいつの会話だっけ？」 が分からず、作業の経過や時系列が把握しづらくなります。

Claude Code本体には showMessageTimestamps という設定がありますが、Cursor拡張機能のチャットUIはこの設定を読みません（CursorがUIを独自に描画しているため）。なので設定を true にしても効きません。

そこで「Claudeに毎回時刻を伝えて、返信の冒頭に文字として書いてもらう」運用にします。UI実装に依存せず、チャット履歴に時刻が確実に残ります。

仕組み（2層構造）

[ユーザーがメッセージ送信] ↓ [Hook層] PowerShellでGet-Date実行 ↓ JSON形式でClaudeのコンテキストに注入 [Claude] 現在時刻を認識 ↓ グローバルCLAUDE.mdのルールに従う [返信冒頭に「YYYY-MM-DD HH:mm (曜日)」を出力] ↓ [チャット履歴に文字として日時が残る]

層1: フック（時刻を毎メッセージ注入）

Claude Codeの UserPromptSubmit フックを使い、ユーザーがメッセージを送信するたびに、現在のシステム時刻をPowerShellで取得してClaudeのコンテキストに渡します。

層2: ルール（返信冒頭に出力させる）

グローバル CLAUDE.md に「返信冒頭に日時を書く」というルールを記述し、Claudeに守らせます。これにより、どのワークスペースで開いたセッションでも同じ挙動になります。

セットアップ手順

所要時間: 5〜10分。Windows環境（PowerShell）を前提に書いています。Mac/Linuxの場合は 注意点セクション を参照してください。

1

settings.json にフックを追加

ファイル: C:\Users\<ユーザー名>\.claude\settings.json

hooks オブジェクトの中に、以下のブロックを追加します。hooks キーが既に存在する場合は中に追記、なければ新規作成してください。

{
  "hooks": {
    "UserPromptSubmit": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "powershell.exe -NoProfile -Command \"[Console]::OutputEncoding = [System.Text.Encoding]::UTF8; @{hookSpecificOutput=@{hookEventName='UserPromptSubmit';additionalContext=('現在の日時: ' + (Get-Date -Format 'yyyy-MM-dd HH:mm:ss (dddd)'))}} | ConvertTo-Json -Compress\"",
            "timeout": 5
          }
        ]
      }
    ]
  }
}

Encoding注意

[Console]::OutputEncoding = [System.Text.Encoding]::UTF8; がないと日本語が文字化けします。PowerShellのデフォルト出力エンコーディングがShift-JISのため、ここでUTF-8に切り替える必要があります。

2

設定を反映する

以下のいずれかでフックを読み込ませます：

  • Claude Codeのチャット内で /hooks を一度開いて閉じる（設定が再読み込みされる）
  • Claude Codeを再起動する
3

グローバルCLAUDE.mdにルールを追加

ファイル: C:\Users\<ユーザー名>\.claude\CLAUDE.md

このファイルは全プロジェクトで読み込まれるグローバルな指示書です。以下のセクションを追加します：

## 返信フォーマット
- 各返信の冒頭に、現在の日時を `YYYY-MM-DD HH:mm (曜日略)` 形式で1行記載する（例: `2026-05-11 15:06 (月)`）
- 装飾記号や絵文字は付けず、テキストのみのシンプルな表記にする
- 日時は UserPromptSubmit フックで毎メッセージ注入される「現在の日時: ...」を参照する
- ツール呼び出しのみで終わるターン（テキスト出力がないターン）はスキップしてよい
- ユーザーから「時刻表示やめて」と言われたら停止する

グローバルCLAUDE.mdが効くわけ

プロジェクト別の CLAUDE.md や記憶ファイル（メモリ）に書くと、そのフォルダで開いたセッションでしか効きません。複数フォルダで使いたい場合は 必ずグローバルCLAUDE.md に書く ようにしてください。

4

動作確認

Claude Codeに何かメッセージを送ってみてください。返信が次のように始まれば成功です：

2026-05-11 15:06 (月)

（ここから返信本文）

日時が出ない場合は次の項目をチェック：

  • settings.json のJSON構文が壊れていないか（カンマの抜けに注意）
  • フックがロードされているか（/hooks を開いて確認できる）
  • グローバルCLAUDE.md が正しい場所にあるか
  • セッションを再起動したか

よくある疑問・注意点

Mac / Linux の場合

PowerShellの代わりにシェルコマンドでフックを書きます。command の中身をたとえば以下のように差し替えてください：

"command": "echo \"{\\\"hookSpecificOutput\\\":{\\\"hookEventName\\\":\\\"UserPromptSubmit\\\",\\\"additionalContext\\\":\\\"現在の日時: $(date '+%Y-%m-%d %H:%M:%S (%a)')\\\"}}\""

表示をやめたいとき

  • 一時的にやめるだけなら、Claudeに「時刻表示やめて」と伝える
  • 完全にオフにするには settings.json の UserPromptSubmit フックを削除＋グローバル CLAUDE.md の該当セクションを削除

秒単位まで表示したい場合

CLAUDE.md のフォーマット指定を YYYY-MM-DD HH:mm:ss (曜日) に変えるだけです。フック側は元から秒まで取得しています。

タイムゾーン

PCのローカル時刻が使われます。海外出張や時差のある作業環境では Get-Date がローカル時刻を返すので、JSTを明示したい場合はフォーマットに K（タイムゾーンオフセット）を追加します。

Cursor内蔵AI（Composer/Chat）には効かない

このフックはClaude Codeプラグイン専用です。Cursor内蔵のAI機能（Cmd+K、Composer、Chat）はAnthropicとは別のAPI経路で動いているため、この設定の影響を受けません。

ワークスペース別の記憶（メモリ）と併用するとき

記憶ファイル（~/.claude/projects/<sanitized-cwd>/memory/）に同じルールを書いても効きますが、ワークスペース単位の効力範囲なので注意。全セッション共通にしたいルールはグローバルCLAUDE.mdへ が原則です。

UserPromptSubmitフックの汎用活用例

このフックは「ユーザーがメッセージを送るたびに、Claudeに渡すコンテキストを動的に追加できる窓口」として汎用的に使えます。日時表示はその一例にすぎません。

応用アイディア

注入する情報 使いどころ
現在のGitブランチ名 「いまどのブランチで作業しているか」をClaudeが自動把握。ブランチ切り替え時の混乱を防げる
カレンダーの次の予定までの時間 「あと20分でMTGなので、簡潔に」とClaude側が配慮できる
バッテリー残量 ノートPCの外出作業時。残量少ないと手短にまとめてくれる
未読Slack件数 割り込み状況の把握。集中作業中なら通知をミュート、など
天気・気温 外出予定の判断、雑談の話題
直近の作業ログ要約 長時間セッションで「さっきまで何やってたっけ」を防ぐ

実装パターン

例えばGitブランチを注入する場合：

"command": "powershell.exe -NoProfile -Command \"[Console]::OutputEncoding = [System.Text.Encoding]::UTF8; $branch = (git branch --show-current 2>$null); if (-not $branch) { $branch = 'no-git' }; @{hookSpecificOutput=@{hookEventName='UserPromptSubmit';additionalContext=('現在のブランチ: ' + $branch)}} | ConvertTo-Json -Compress\""

複数の情報を1つのフックでまとめて渡す

フックを複数登録する代わりに、1つのフック内で複数の情報をまとめて additionalContext に入れる書き方が推奨です。フックが多いとセッション起動・各メッセージごとに余分なプロセスが立ち上がるため。

$dt = Get-Date -Format 'yyyy-MM-dd HH:mm (dddd)'
$branch = (git branch --show-current 2>$null)
$ctx = "現在の日時: $dt`n現在のブランチ: $branch"
@{hookSpecificOutput=@{hookEventName='UserPromptSubmit';additionalContext=$ctx}} | ConvertTo-Json -Compress

他のフックイベント

UserPromptSubmit 以外にも、Claude Codeにはさまざまなフックイベントがあります：

イベント 発火タイミング 用途例
PreToolUse ツール実行前 危険コマンドをブロック、ログ記録
PostToolUse ツール実行後 編集後にフォーマッタ自動実行、テスト走らせる
Stop Claudeが返答完了したとき 音を鳴らす、Slackに通知
SessionStart セッション開始時 初期コンテキスト注入、環境チェック
PermissionRequest 権限要求時 音を鳴らして気付かせる

運用上の注意

フックは 毎メッセージ／毎ツール実行ごと に走るので、重い処理を書くと体感速度が落ちます。timeout を短め（5〜10秒）に設定し、ネットワーク呼び出しなどブロッキングする処理は避けましょう。
