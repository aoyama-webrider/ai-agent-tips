Claude Code Hook Stop PermissionRequest

Claude Codeの「完了」「許可待ち」を音で知らせる

想定環境: Windows ＋ Claude Code拡張（VS Code等）＋ Claudeサブスク

なぜ音通知が必要なのか

Claude Codeに少し重めの作業を投げると、完了まで数十秒〜数分かかることがあります。その待ち時間に別の画面で作業していると、「終わったことに気づかず放置」「許可を求められて止まっているのに気づかず時間をロス」 ということが起こりがちです。

そこで、Claude Codeが 応答を終えたとき と コマンド実行などの許可を求めてきたとき に通知音を鳴らすようにします。ながら作業をしていても、音だけで「いま自分の番が来た」と分かるようになります。

この記事で使うフック

Stop（Claudeが返答を完了したとき）と PermissionRequest（権限の許可を求めてきたとき）の2つです。完了音と許可待ち音で別々の音を割り当てると、振り向かなくても状況が判別できて便利です。

仕組み

Claude Codeのフックに、音声ファイルを再生するコマンドを登録するだけです。Windowsなら標準搭載のPowerShellで、OS同梱のwavファイルを鳴らせるので、追加インストールは不要です。

[Claudeが返答を完了] ↓ [Stopフック発火] PowerShellでchimes.wavを再生 → 「完了したよ」の音 [Claudeがコマンド実行の許可を要求] ↓ [PermissionRequestフック発火] notify.wavを再生 → 「確認して」の音

ポイントは、再生に PlaySync() を使うこと。フックは短命なサブプロセスとして起動するため、非同期再生（Play()）だと音が鳴り切る前にプロセスが終了して無音になることがあります。PlaySync() は再生完了まで待つので確実です。

セットアップ手順

最短ルート: AIエージェントに任せる

手作業が不安なら、Claude CodeにこのページのURLを渡して、次のように頼むだけでOKです。設定ファイルの編集はAIがやってくれます。

https://aoyama-webrider.github.io/ai-agent-tips/sound/
↑このページのとおり、私の settings.json に通知音の設定を反映して。
・既存の hooks 設定は壊さず、マージして（上書きしない）
・終わったら、反映に再読み込み（/hooks を開き直す or 再起動）が必要か教えて

想定環境: Windows ＋ VS Codeなどのエディタ ＋ Claude Code拡張 ＋ Claudeサブスク。この前提なら、ほぼそのまま再現できます。ただし最後の再読み込み（/hooks を開き直す or 再起動）だけは手動です（AIはエディタの再起動まではできません）。

仕組みを理解しながら自分で設定したい人向けに、手順も載せておきます（所要時間 約5分・Windows/PowerShell前提）。Mac/Linuxの場合は 注意点セクション を参照してください。

1

settings.json にフックを追加

ファイル: C:\Users\<ユーザー名>\.claude\settings.json

hooks オブジェクトの中に、以下の Stop と PermissionRequest のブロックを追加します。すでに hooks がある場合は中身を消さずにマージしてください（例: 日時表示 の UserPromptSubmit など他のフックがあるときは、それらを残したまま Stop / PermissionRequest を足す）。

{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "powershell.exe -Command \"(New-Object Media.SoundPlayer 'C:\\Windows\\Media\\chimes.wav').PlaySync()\""
          }
        ]
      }
    ],
    "PermissionRequest": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "powershell.exe -Command \"(New-Object Media.SoundPlayer 'C:\\Windows\\Media\\notify.wav').PlaySync()\""
          }
        ]
      }
    ]
  }
}

完了音だけでいいなら

まずは Stop だけ入れてみるのがおすすめです。許可待ちにも気づきたくなったら PermissionRequest を足す、という順で増やすと音の鳴り方を体で覚えられます。

2

使えるWindows標準サウンドを選ぶ

C:\Windows\Media\ フォルダにOS標準のwavファイルが入っています。エクスプローラで開いてダブルクリックすると試聴できます。よく使われるもの：

ファイル名 音のイメージ
chimes.wav やわらかいチャイム。完了通知向き
notify.wav 短い通知音。許可待ち向き
ding.wav 軽い「ポーン」。控えめにしたいとき
tada.wav ファンファーレ。完了をはっきり知らせたいとき
Windows Notify System Generic.wav 標準の通知音。Windowsらしい音

パスをそのまま command の中の 'C:\\Windows\\Media\\〇〇.wav' 部分に差し替えればOKです（JSON内なのでバックスラッシュは \\ と2個重ねる点に注意）。

3

設定を反映する

以下のいずれかでフックを読み込ませます：

  • Claude Codeのチャット内で /hooks を一度開いて閉じる（設定が再読み込みされる）
  • Claude Codeを再起動する
4

動作確認

Claudeに簡単な質問を1つ投げてみてください。返答が終わった瞬間に Stop の音（chimes）が鳴れば成功です。次にコマンド実行を伴う依頼（例: 「いまのフォルダのファイル一覧を出して」）をすると、許可ダイアログが出るタイミングで PermissionRequest の音（notify）が鳴ります。

鳴らない場合は次の項目をチェック：

  • settings.json のJSON構文が壊れていないか（カンマの抜け、バックスラッシュの数に注意）
  • フックがロードされているか（/hooks で確認できる）
  • 指定したwavファイルが実際に C:\Windows\Media\ に存在するか
  • PC本体やアプリの音量がミュートになっていないか
  • セッションを再起動したか

よくある疑問・注意点

音が鳴りすぎてうるさいとき

  • まず PermissionRequest を外して Stop だけにする
  • 権限の許可プロンプト自体が多い場合は、よく使う安全なコマンドを settings.json の permissions.allow に登録しておくと、そもそも許可要求＝音が減ります
  • もっと控えめな音（ding.wav など）に変える

自前の音源を鳴らしたい

任意の .wav ファイルのパスを指定すれば使えます。SoundPlayer はwav形式のみ対応で、mp3は再生できない点に注意。mp3を使いたい場合は (New-Object Media.SoundPlayer ...) の代わりに以下のようにWindows Media Player経由で鳴らせます：

"command": "powershell.exe -NoProfile -Command \"Add-Type -AssemblyName presentationCore; $p = New-Object System.Windows.Media.MediaPlayer; $p.Open([uri]'C:\\path\\to\\sound.mp3'); $p.Play(); Start-Sleep -Seconds 2\""

音量を調整したい

SoundPlayer 自体には音量指定がありません。OSのアプリ音量ミキサーでPowerShellの音量を下げるか、あらかじめ音量を絞ったwavファイルを用意してください。

Mac / Linux の場合

PowerShellの代わりにOS標準の再生コマンドを使います。

macOS（afplay が標準搭載）：

"command": "afplay /System/Library/Sounds/Glass.aiff"

Linux（paplay や aplay など、環境にあるものを使う）：

"command": "paplay /usr/share/sounds/freedesktop/stereo/complete.oga"

止めたいとき

  • 一時的に消したいだけなら、PCやアプリの音量をミュートする
  • 完全にオフにするには settings.json の Stop / PermissionRequest フックのブロックを削除する

処理が遅くならない?

音の再生は一瞬で終わるうえ、Stop はメッセージごとに1回・PermissionRequest は許可要求時のみなので、体感への影響はほぼありません。ただしフックの中でネットワーク通信など重い処理を足すと遅延の原因になるので、音を鳴らすだけに留めるのが安全です。

音以外の通知方法

同じ Stop / PermissionRequest フックの仕組みで、音以外の通知もできます。集中作業中や、席を外しているときに気づきたいケースで使い分けると便利です。

通知方法 使いどころ
Windowsのトースト通知 音を出せない環境（会議中・イヤホンなし）。画面右下にポップアップ
Slack / Discordへ通知 席を外していてもスマホで気づける。Webhook URLにPOSTするだけ
音声読み上げ 「完了しました」と喋らせる。画面を見ていなくても内容が分かる

例: 完了時にWindowsトースト通知を出す

音の代わり（または併用）に、画面右下へポップアップを出すパターン：

"command": "powershell.exe -NoProfile -Command \"[Console]::OutputEncoding=[Text.Encoding]::UTF8; New-BurntToastNotification -Text 'Claude Code', '応答が完了しました'\""

事前準備が必要

上の New-BurntToastNotification はモジュール BurntToast のインストール（Install-Module BurntToast）が前提です。標準機能だけで完結させたい場合は、まず本記事の音通知から始めるのがおすすめです。

例: 完了時に音声で読み上げる

"command": "powershell.exe -NoProfile -Command \"Add-Type -AssemblyName System.Speech; (New-Object System.Speech.Synthesis.SpeechSynthesizer).Speak('完了しました')\""

フックの全体像をもっと知りたい人へ

フックには Stop / PermissionRequest のほかにも、ツール実行前後（PreToolUse / PostToolUse）やメッセージ送信時（UserPromptSubmit）などがあります。日時表示のTip の後半で各イベントの一覧と活用例をまとめているので、あわせてどうぞ。
